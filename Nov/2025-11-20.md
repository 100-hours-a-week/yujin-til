### 🚀 Today I Learned

#### Docker Compose

- 여러 컨테이너로 구성된 애플리케이션을 단일 명령어로 동시에 실행/관리할 수 있는 도구
- 컨테이너 가상화 → 많이 만들어질수밖에 없음 → 왜? 격리할라고 → 근데 컨테이너 하나 구상할때마다 d run할 수 없으니 → 코드로 만들자 → YAML 형식의 파일로 정의함
- docker compose 사용하면, 기본으로 같은 네트워크로 공유함
- 사용 이유 : 다수의 컨테이너로 구성된 애플리케이션을 코드로 명확히 정의하고, 한번에 쉽개 관리하기 위해
- 개발환경 분리할때 많이 사용함
  - 예를들어
    - local 에서도 돌아가야하고
    - dev 서버 ← ubuntu에서도 돌아가야하고
    - staging 서버 ← ec2 2개, RDS, 엘라스틱캐시 에서도 돌아가야하고
    - production 서버에서도 돌아가야함
    - Dockerfile도 분리시킬수있음
      - Dockerfile ← 로컬
      - Dockerfile.prod ← 운영용
        - 라이브러리 더 줄이거나, 환경변수를 클라우드에서 가져오거나
  - EC2 기반 : docker-compose.yml, docker-compose.dev.yml, docker-cmoopse.prod
  - ECS 기반 ← 관리형 컨테이너 서비스
    - 컨테이너를 올리면(도커 파일, 이미지 올리면)
    - 그 이미지로 컨테이너를 만들고, 그 컨테이너를 통해 서비스함
  - 어떤걸 선택하느냐는 필요에 따라 선택. 혼용해서도 가능
- docker compose up 전에 docker compose ps 하면 제대로 동작할수있는지 확인 가능
