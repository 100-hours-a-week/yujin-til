### 🚀 Today I Learned

- 데이터베이스
  - 구조화된 정보(데이터)들의 모음
  - Why? 데이터를 안전하고 편리하게 보관/사용하기 위해
- 관계형 데이터베이스(RDB)
  - 구조화된 정보(데이터)들의 관계 모음
- RDBMS
  - 관계형 데이터베이스를 관리하는 시스템
- Query
  - 데이터베이스에서 원하는 데이터를 찾고/삽입/삭제 등을 할 수 있게 하는 질문
- SQL (DDL, DML, DCL)
  - 데이터베이스에서 원하는 자료를 찾고/삽입/삭제하기 위해 사용하는 언어
  - DDL(데이터 정의어) : DB의 구조를 만들고 수정/삭제하는데 사용하는 명령어 ex) Create.Alter
  - DML(데이터 조작어) : DB의 데이터를 조회,추가 등 조작할 수 있는 명령어 ex) Insert, Update,Select
  - DCL(데이터 제어어) : DB의 권한을 사용자에게 부여/회수할 수 있는 명령어 ex) grant,revoke
- ERD
  - 개체와 그 개체들의 관계를 적어놓은 다이어그램
  - Why? DB를 시각적으로 표현하여 알아보기 쉽게 하기 위해 사용
  - 기획서 등 자료 → 개념적 모델링 → 논리적 모델링 → DB로 구현(물리적 모델링)
- Index
  - DB에서의 조회 속도를 높이기 위해 사용하는 자료구조
  - LIKE ‘%검색어’ - 인덱스 안탐
    - % 때문에 정렬이 어려워서 인덱스로 걸러내가 어려움
- Cardinality
  - 상대적인 중복의 정도
  - **[최적화(DB) 기반] distinct 값 많다? → 중복도가 낮다 → Cardinality가 높다**
  - 이거 기준으로 index생각하면 될듯(Selectivity는 파생된 개념)
- Selectivity
  - 선택도
  - 특정 컬럼이 얼마나 선택되는지를 나타내는 척도
  - [최적화(DB) 기반] index는 많이 걸러낼 수록 유리 = 조건을 만족하는 행이 적을수록 유리 = 선택도가 낮을수록 → 인덱스로 적합
  - **[컬럼 자체의 Selectivity] Cardinality 높을수록 → 선택도가 높다 → 인덱스로 적합**
- Tree (binary tree, b-tree, b+tree)
  - 노드로 이루어진 계층적인 데이터 구조
  - binary tree : 노드 하나당 최대 2개의 노드로 구성되는 이진법 트리구조
    - 정렬이 가능한 값을 기준으로 만들어짐
    - But, 한쪽으로만 노드가 길어질 수 있는 편향문제가 존재 ⇒ 이를 위해 Balance가 추가됨
  - b-tree
    - 하나의 노드가 여러개의 노드 가질수 있음. 트리의 높이를 균형있게 유지가능
    - But, 이 균형을 맞추기 위한 비용 발생 & 범위 검색이 어려움 ⇒ b+tree 등장
  - b+tree
    - b-tree의 확장 개념
    - 브랜치 노드 - key만 담음(인덱스 역할만), 리프 노드- key, data 저장, 리프노드 끼리는 Linked list로 연결
    - 따라서, 메모리 확보 가능 → 더 많은 key 수용 가능 → 트리 높이 더 낮아짐
    - ex) mysql innodb
- Inverted Index
  - 단어를 key로 삼고, 이 단어를 포함하는 문서들을 값으로 갖는 index
- Full text Index
  - 정렬할 수 없는 값(ex.문장)을 의미있는 단위로 쪼개서 검색하는 기법
  - 내부적으로 Inverted Index 사용
    - Full text Index만들면 DB가 텍스트를 자동으로 단어 단위로 쪼개서 Inverted Index에 저장함
- Transaction
  - 한번에 수행할 연산들의 묶음 → 하나의 기능을 구현하기 위해 사용해야하는 쿼리들의 묶음?
  - Why? 여러 작업을 하나의 논리적인 작업단위로 묶어서 처리하기 위함
- ACID
  - 트랜잭션의 특징으로, 모두 수행하거나 않거나(원자성), 결과가 일관성을 보장하고(일관성), 트랜잭션이 독립적으로 동작하고(독립성), 트랜잭션이 성공적으로 완료됐다면 결과는 영구적으로 반영되어 있어야 한다(영구성).
- Isolation Level
  - 트랜잭션이 얼마나 독립되어있는지를 나타내는 정도
  - L0~L3까지 존재
    - 무조건 높다고 좋은것은 아님
