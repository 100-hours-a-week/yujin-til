### 🚀 Today I Learned

- AWS Lambda + API Gateway 조합으로 이미지 업로드 서버리스 구조를 직접 구현해보고, 프론트와 백엔드 양쪽에 반영해보면서 전체 흐름을 이해함.

- 하지만 MPA 방식에서는 Lambda URL이 클라이언트에 노출될 수밖에 없고, 인증을 붙이지 않으면 누구나 파일을 업로드할 수 있는 보안 위험이 발생한다는 사실을 명확히 깨달음.

- Lambda 기반 업로드를 안전하게 쓰려면 API Gateway Authorizer(JWT 검증) 같은 인증 계층이 반드시 필요하다는 점을 배움.

- 결국 “프로젝트의 규모와 구조(MPA)”에 맞춰, Lambda가 아닌 백엔드(S3 direct upload) 방식이 더 적합할 수 있다고해서 진행할 예정
  - Lambda를 그대로 쓰려면 Authorizer 구성, 인증 연결 등 관리 포인트가 많아지고, 잘못되면 누구나 업로드가 가능하기에 이미지를 서버가 업로드하는 방식으로 바꾸는게 보안 + 성능 + 개발 편의성 관점에서 현재 프로젝트 구조에 가장 적합한 선택이라고 판단함

### 🗨️ 오늘의 회고

- Lambda를 구현하는 과정에서 “서버리스 구조는 이런 흐름으로 동작하는구나”를 느꼈다. 하지만 동시에 보안 문제, 인증 구조, MPA 프론트의 한계 등 실제 서비스 환경에서의 안정성 같은 요소들을 고민하면서 내 서비스에는 뭐가 나을까를 고민하게 되었다. 결국 오늘 했던게 쓰이진 않지만, 그래도 좋은 경험했다고 생각하기로 했다..
